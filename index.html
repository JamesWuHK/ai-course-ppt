📚 基础概念

什么是 AI 和大模型？

🧠

会理解语言

你像和人说话一样下指令，它能理解你的问题、要求和上下文，再给出回答或草稿。

⏰

会生成内容

它可以根据你的要求生成文字、图片、音乐、视频等内容，所以也叫“大模型”。

💬

会给第一版

它最适合先帮你产出第一版内容，再由你挑选、修改、审核，而不是直接替你拍板。

豆包
聊天、问答、文案起草
Kimi
长文阅读、总结整理
通义千问
问答、办公、写作
文心一言
问答、写作、助手类
即梦
图片、视频生成
对小白来说，先记住一句话就够了：大模型本质上是一个会听懂你要求、并且能生成内容的助手。先别急着学 Agent，先把它最基础的能力看明白。

⚙️ 基础功能

大模型最常见的基础功能

出文案

最常见的能力就是写文字。比如招生朋友圈、家长通知、课程介绍、短视频口播稿、活动标题。

出图片

可以根据一句话生成海报图、封面图、插图，也可以把现有图片改风格、补背景、换构图。

出音乐

可以按主题和氛围生成背景音乐、儿歌 demo、轻配乐，适合做宣传视频或课堂氛围素材。

出视频

像即梦这类工具，可以做文生视频、图生视频、短片段演示，适合做宣传短视频或视觉样片。

这一步先不要想复杂。对小白来说，大模型最基础的能力就是四类：写文字、出图片、出音乐、出视频。先理解它会“产出内容”，后面再理解它怎么“调用工具做事”。

国内常见 AI 工具，先这样理解

豆包

更适合小白先上手。常见用法是聊天问答、改文案、出活动标题、写通知、帮你把想法变成第一版。

适合先用来练“怎么把需求说清楚”。

Kimi / 通义 / 文心

这类工具的共同点是：问答、总结、读资料、写内容。不同工具风格不同，但对小白来说先把它们当成“内容助手”就够了。

适合先处理文档、方案、总结、资料阅读。

即梦

即梦常见的基础能力是出图和出视频。适合做宣传视觉、活动海报样图、短视频片段和创意演示。

适合先做视觉内容，而不是流程自动化。

音乐生成功能

常见用法是按主题、节奏、情绪出 BGM、儿歌 demo、短音频片段，适合做宣传视频或课堂氛围素材。

适合配内容，不适合一开始就追求专业成品。

图片生成功能

一句话生成图片、改图、换背景、延展比例、做海报主视觉，适合给运营和招生快速出样。

先拿来做“第一版”，再人工细修最稳。

视频生成功能

可以做短片段、镜头样片、动态封面或口播配图视频。对于小白，先把它当成“宣传素材生成器”更容易上手。

先做短视频素材，不要一开始就想做完整大片。

小白一开始最容易混淆的是“工具很多，不知道该从哪里开始”。先记住：聊天问答和文案类工具，主要帮你产出内容；即梦这类工具，主要帮你产出视觉；这些都还属于大模型的基础功能。

⚡ 关键区别

什么时候是“大模型”，什么时候是“Agent”？

🤖

大模型基础功能

先产出内容

  • 💬 聊天问答、写文案、做总结、出图片、出音乐、出视频
  • 📝 更像一个内容助手，帮你给出第一版
  • 👀 你通常需要自己再去复制、发送、整理、执行

举个例子

你说：“帮我写 3 条寒假班招生文案” → 大模型给你 3 个版本

VS
🚀

Agent

继续调用工具做事

  • ⚙️ 不只是生成内容，还会继续调用文档、表格、文件、浏览器等工具
  • 🧩 更像一个执行助手，会把任务拆成步骤推进
  • ✅ 适合整理数据、归档资料、跨工具协同，但高风险动作仍要人工确认

举个例子

你说：“把本周试听、到课和转化数据整理成复盘表并生成总结” → Agent读取数据、整理表格、生成结论

到这里就能看出差别了：大模型先负责“产出内容”，Agent 再负责“调用工具继续把任务往前做”。这两个不是对立关系，而是前后衔接关系。

教培机构 AI 应用拆解

以下为课堂示意案例，用来帮助你判断“什么任务值得先试”。

🏫

K12学科培训机构

痛点

家长咨询重复问题多（课程价格、上课时间、教学效果），客服人手不足，响应慢导致流失严重

AI解决方案

AI先完成首轮 FAQ 回复、试听信息收集和意向标签整理，再由课程顾问接力完成转化

AI提示词

"你是机构的课程顾问助理，请先回答班型和试听安排；涉及价格和优惠时提醒转人工确认。"

📈

示意收益：首轮回复更稳定，课程顾问把时间留给高意向家长

📚

职业技能培训班

痛点

学员学习进度参差不齐，老师无法一对一回访，无法及时发现掉队学员，完课率只有45%

AI解决方案

AI自动整理学习数据、生成掉队预警清单和回访草稿，老师据此安排重点跟进

AI提示词

"分析本周学习数据，识别近两周活跃度下滑的学员，并给出老师回访提纲。"

🎯

示意收益：老师更快发现掉队风险，回访动作更及时

🎨

少儿兴趣培训机构

痛点

家长总问"孩子学得怎么样"，老师课后要花大量时间写个性化反馈，消耗精力还家长不满意

AI解决方案

AI先整理课堂记录、生成家长汇报草稿和阶段建议，由老师补充关键观察后再发送

AI提示词

"根据这学期课堂记录，生成一份给家长的成长汇报草稿，突出进步、作品亮点和下阶段练习建议。"

😍

示意收益：课后反馈更完整，家长沟通口径更统一

⚠️ 风险边界

哪些任务必须人工复核？

⚠️ 必须人工确认的内容

价格、优惠、退费政策
升学承诺、提分承诺、结果判断
个体化学习建议、健康安全相关提醒

原因：这些内容一旦说错，影响的是机构信誉、合规风险和家长信任。

要求：AI 可以先起草，但最后一定要由老师、顾问或校长确认。

✅ 更稳的协作方式

AI 先整理资料和起草
人来确认口径和边界
最终再发送或执行

核心：把 AI 放在“提速”和“打底”的位置，而不是完全替你拍板。

结果：既省时间，也保留专业判断和信任感。

这一页最重要的结论是：只要涉及学员隐私、家长承诺、价格政策、学习判断，AI 只能协助提速，最终判断必须由人负责。
直播判断题：在你机构里，哪些内容一旦说错，影响最大？把那一类全部划进“必须人工复核”。

🛠️ 方法模板

给学员带走的最小指令模板

先会这一个模板就够了

对象 + 任务 + 约束 + 输出格式

1

1. 对象

你要面对谁：家长、老师、校长、顾问、学员

2

2. 任务

你想让 AI 做什么：起草、整理、分析、归纳、改写

3

3. 约束

写清边界：不要承诺价格、不要碰隐私、语气要温和专业

4

4. 输出

最后要什么：三版文案、一页表格、一段回复、一个提纲

🎯 你可以直接这样说：

"帮我为三到六年级家长起草一条寒假班试听通知，要求语气专业、不过度营销，不要写死价格和优惠，最后输出3 个可直接修改的版本。"

✓ 先说清“写给谁”
✓ 再说“要做什么”
✓ 明确“不能做什么”
✓ 最后指定“交付格式”
大多数教培场景并不需要复杂提示词。先记住一个最小模板：对象、任务、约束、输出格式，就足以覆盖大部分起稿、整理和分析类任务。
