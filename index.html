历史一笔带过

1950 年，图灵提出一个有名的问题：如果机器聊天聊到让人分不清对面是不是人，算不算“会思考”？这就是后来常说的图灵测试。你可以把它理解成：看机器能不能通过对话，让人误以为对面是真人。 图灵测试可以先这样理解 左边是人，右边是机器人，中间隔开，双方只能通过文字交流。如果你聊了一阵还分不清对面是谁，这个测试就算它过关。

老一代 AI

过去很多 AI 更像“规则表 + 人工经验”。典型例子就是早期人脸识别：抓五官的形状、位置和彼此之间的距离，再把这些特征提取出来做识别。 早期人脸识别就是典型的“先提特征” 左边先把眼睛、鼻子、嘴巴的位置和距离标出来，中间整理成可计算的特征，右边再拿这些特征去做识别。它很实用，但灵活性不够。

今天的大模型

现在这类大模型，会先吃进海量数据，自己学规律。过去更像先把规则写好；现在更像先大量学习，再根据你的问题现场作答。

DeepSeek 这类工具适合先做什么

DeepSeek 这类工具，更像一个会文字交流的智慧大脑。它能陪你讨论工作、分析问题、梳理思路、提供建议，也能写文字内容。它强在思考和交流；如果要直接替你操作电脑、跑流程，那就要看后面的 Agent 工具了。

第一页实验只做一件事：让大家亲手感受到“AI 会听懂我的话，也能顺着我的问题继续一起想”。先有这个体感，后面的新概念才会长在体验上。
当堂操作：写一句真实任务，先看 AI 怎么回应，再补 1 轮要求。

先把“大模型”这个脑子理解清楚

产品截图 · DeepSeek

在同一个对话里连续追问 3 轮，更容易理解“大模型脑子”是怎么工作的

不用切到别的软件，就用刚才那条对话继续改，看它怎样一轮轮贴近你的真实业务要求。

第 1 步：沿用上一条对话 第 2 步：连续补对象、场景、限制 第 3 步：观察答案一轮比一轮更贴合
当堂练习

同一条需求，连续改 3 轮

  • 不新开会话，直接继续追问。
  • 第 1 轮改对象和口吻，第 2 轮补时间地点，第 3 轮加边界和输出格式。
  • 观察一下：为什么越追问，结果越贴近真实场景？
关键是它能记住前文，并按你的新要求继续修改。
照着做
1

不要新开会话，沿用刚才那条对话继续追问。

2

第 1 轮改对象和口吻，让它更贴近你真实受众。

3

第 2、3 轮再补限制和输出，看它能不能顺着上下文继续改。

看到什么算完成
✓

完成标准：每追问一轮，结果都更贴业务，而不是前后断掉重来。

!

追问四件套：对象、场景、限制、输出。追不准时先补这 4 样。

1. 特征提取

特征提取，就是先把文字、图片变成机器能计算的信号。也就是先把“人话”和“画面”变成机器能处理的格式，它后面才有办法学习和判断。

2. 学习

训练时，模型会看数千亿到万亿级 token 的语料。`token` 你先把它理解成模型切出来的“小块”就行，它不完全等于一个字，有时是一个词，有时是半个词。模型就是在海量样本里反复找规律。

3. 推理

你现在发问题时，模型并不是临时再上学，而是在用已经学到的参数做推理和预测，算出下一句最可能怎么接、怎样更符合你的要求。

大模型到底在做什么

不管是 DeepSeek、豆包，还是后面的 Agent 软件，底层都是这个“大模型脑子”。它先学了很多，再根据你眼前的问题来思考和作答。你把对象、场景、限制说得越清楚，它给出的结果就越贴业务；但最后怎么定，还是要人来定。

这页主要是把刚才的体验说清楚。学员不需要背定义，只要知道：DeepSeek、豆包这些工具背后，其实都是大模型。
用自己的话说一句：为什么刚才那个实验是在用大模型？

豆包这类工具能做什么

豆包这类工具，比第一步更进一步。它不只会文字交流，还会看图、听音、理解不同类型的信息，这叫多模态；它还能生成图片、视频这些新内容，这叫 AIGC。对机构来说，这一步已经直接碰到海报、视频、课程包装、活动物料这些事了。

这里先做两件事：先体验豆包“看图 + 追问”，理解什么叫多模态；再看 AI 生成图片和视频，理解什么叫 AIGC。
当堂操作：上传一张素材并追问一次，再判断这是多模态还是 AIGC。

故事主视觉

先记住这个形象：一只半生物、半机械的大龙虾

这个形象不是为了猎奇。它把 OpenClaw 想表达的方向画出来了：不是只会聊天的 AI，而是围绕 Agent 智能体想象出来的新物种。

故事线 1：它不是突然冒出来的爆款，而是一条一路迭代、一路长能力的产品线。
故事线 2：它一下子火起来，不只是名字特别，更因为很多人第一次认真注意到 Agent 智能体这条路线。
官方故事页

它不是突然冒出来的，而是一路演化过来的

从官方故事页能看到，它最早从 Warelay 这样的 WhatsApp gateway 起步，后来随着 AI 角色和能力不断发展，先后用过 Clawd、Molty 等名字。到了 2026 年 1 月 27 日，团队因为商标问题收到改名邮件，最后在 2026 年 1 月 30 日 定名成 OpenClaw。它更像是一条逐步长出来的路线，而不是突然蹿红的新名字。

人物和时间线

一个产品为什么能带火一条路线，也和做产品的人有关

Peter Steinberger（@steipete） 是 OpenClaw 的 creator。公开资料显示，他此前是 PSPDFKit 的创始人，长期做原生 iOS / macOS 开发和工具产品，后来持续投入 AI 工具和开源生态。这个背景很重要，因为 OpenClaw 从一开始就不是单纯做聊天，而是奔着 Agent 智能体这条产品路线去的。

Agent 它不是普通聊天框。Agent 智能体的关键特点，是能根据目标自主决定调用工具，把任务往下推进
Soul Memory 把重要的人和事记下来。下次再聊时，不用每次从头交代，智能体还能接着前面的上下文做事
Skill 把一套已经跑顺的操作整理成可复用步骤。下次遇到同类任务，可以直接调用，不用重新描述一遍

Agent 和普通聊天工具差在哪

这类 Agent 智能体，不只是会聊天的 AI。它们是建立在大模型之上的新一层软件，特点是能根据目标自主决定调用工具，所以不只会思考、会多模态交互，还能直接用电脑干活，替人做掉一部分重复工作。再往前走一步，就是把常做操作整理成 skill，让经验能留下来，后面继续用。 “养龙虾”不是多一个聊天框 就是把你已经跑顺的操作步骤整理成 skill。下次再做同类任务，不用从头描述，直接调用就行。

说到这里，“养龙虾”其实就很简单：不是让 AI 临时帮一次，而是把你已经验证过的日常操作整理成 skill，后面反复调用。
你会怎么补这句话：AIGC 是先帮我生成，Agent 是开始帮我把事情 ______。

WorkBuddy 已经能做什么

WorkBuddy 这类工具的关键，不是多了一个聊天框，而是它作为 Agent 智能体，能根据目标自主决定调用工具，再配合 Computer Use 去完成动作。所以它不只是帮你出主意，而是真的可以接住查资料、开网页、整理文件、产出结果这些具体动作，替人做掉一部分工作。 Computer Use 可以这样理解 不是 AI 说“你可以去做什么”，而是它自己去开网页、读文档、整理表格，再把结果放回给你检查。

这里把最后一个词讲清楚：Computer Use。学员不需要背技术定义，只要明白一件事就够了：当 AI 开始真的去动电脑、动文件、动网页，它就是在替你做电脑上的动作。
试着把 6 个词串起来：AI、大模型、多模态、AIGC、Agent、Computer Use（让 AI 动电脑）。你最想先试哪一层？
