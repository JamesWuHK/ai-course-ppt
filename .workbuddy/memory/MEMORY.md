## 课程设计原则（用户明确提出）
1. **不教零散GUI工具**，数字员工/Agent已覆盖这些能力
2. **反对手动编排流程**（如Coze），这是Agent的工作不是人的工作
3. **三大企业IM均已开放CLI**：飞书(lark-cli)、钉钉(dingtalk-workspace-cli)、企微CLI均已开源，Agent可直接操控
4. **海报Skill**：直接描述需求出图，无需描述画面细节
5. 核心理念：人说需求 → Agent自动调用Skill → 完成任务
6. **课程不依赖单一平台**：飞书/钉钉/企微三大平台均已支持，根据学员实际使用平台举例

## ⚠️ 课程设计关键前提（2026-03-30新增）
- **0基础**：学员完全不懂AI，可能从未用过任何AI工具
- **陌生客户**：不知道学员的AI认知程度
- **从最基础讲起**：不能假设任何先验知识
- **全程互动**：随时了解反馈，根据反馈调整讲解深度

## 课程互动环节设计（v4.3新增）
1. **课前破冰（3分钟）**：举手统计AI使用经验，根据反馈调整深度
2. **课间小调查**：每阶段结束后检查有没有听不懂的
3. **现场互动"你来说，我来做"**：邀请学员说困扰，现场演示Agent解决
4. **课末反馈收集**：每人说一个回去想试的场景
5. **快速小测**：用简单问题确认核心概念已掌握

## 线上演讲技巧（v4.4新增）
